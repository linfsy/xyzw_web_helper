# 系统稳定性优化方案

## 文档目标
本文档详细描述系统稳定性优化方案，确保积攒列表、暂停时间、任务持久化和分批执行功能不出问题。

## 系统现状分析

### 存在的问题
1. **导入配置需要两次**：定时任务存储键名不匹配，导致第一次导入时定时任务不显示
2. **积攒队列管理问题**：
   - 单账号任务被跳过
   - 任务去重逻辑不完善
   - 数据加载/保存错误时清空数据
   - 一个任务完成后清空整个队列
3. **暂停时间处理**：批量执行等待期间未检查暂停状态
4. **任务持久化**：状态管理和恢复逻辑需要优化
5. **分批执行**：批处理配置和错误处理需要完善

## 核心优化策略

### 1. 积攒列表优化

#### 1.1 智能任务去重
- **实现要点**：同时检查任务名称、类型、账号列表和任务类型
- **具体逻辑**：
  - 检查任务名称是否相同
  - 检查运行类型是否相同
  - 检查选中的账号列表是否相同
  - 检查选中的任务类型是否相同
- **应用位置**：
  - `batchTaskStore.ts` 中的 `addToTaskQueue` 函数
  - `BatchDailyTasks.vue` 中的任务添加逻辑

#### 1.2 安全队列管理
- **实现要点**：成功的任务逐个移除，失败的任务保留
- **具体逻辑**：
  - 任务执行成功后，从队列中移除该任务
  - 任务执行失败后，保留在队列中
  - 避免统一清空队列的操作
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `executeQueuedTasks` 函数

#### 1.3 错误处理增强
- **实现要点**：数据加载/保存失败时保留现有数据
- **具体逻辑**：
  - 加载失败时，保留现有队列数据
  - 保存失败时，保留现有队列数据
  - 添加错误日志，便于排查问题
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `loadTaskQueueFromStorage` 函数
  - `BatchDailyTasks.vue` 中的 `saveTaskQueueToStorage` 函数

#### 1.4 队列长度限制
- **实现要点**：限制队列长度，防止存储过大
- **具体逻辑**：
  - 最多保存50个任务
  - 超出限制时，保留最新的任务
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `saveTaskQueueToStorage` 函数

### 2. 暂停时间优化

#### 2.1 实时状态检测
- **实现要点**：定时检查暂停状态，确保及时响应
- **具体逻辑**：
  - 每分钟检查一次暂停状态
  - 批量执行等待期间也检查暂停状态
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `updatePauseStatus` 函数
  - `BatchDailyTasks.vue` 中的批处理等待逻辑

#### 2.2 优雅任务暂停
- **实现要点**：暂停时将任务加入积攒队列，保留执行状态
- **具体逻辑**：
  - 检测到暂停状态时，将当前任务加入积攒队列
  - 保留任务的所有配置信息
  - 暂停时间结束后自动恢复执行
- **应用位置**：
  - `BatchDailyTasks.vue` 中的任务执行逻辑
  - `BatchDailyTasks.vue` 中的定时任务调度器

#### 2.3 智能恢复机制
- **实现要点**：暂停时间结束后自动恢复执行
- **具体逻辑**：
  - 定期检查暂停状态
  - 暂停时间结束后，自动开始执行积攒队列中的任务
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `startResumeCheck` 函数

### 3. 任务持久化优化

#### 3.1 安全存储
- **实现要点**：使用 safeLocalStorage 确保存储操作的安全性
- **具体逻辑**：
  - 所有本地存储操作都使用 safeLocalStorage 包装
  - 处理存储满、权限不足等异常情况
- **应用位置**：
  - 所有本地存储操作

#### 3.2 状态管理
- **实现要点**：正确保存和恢复任务状态
- **具体逻辑**：
  - 保存运行中任务的状态
  - 页面刷新后恢复任务状态
  - 任务完成后清除状态
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `saveRunningTask` 函数
  - `BatchDailyTasks.vue` 中的 `clearRunningTask` 函数

#### 3.3 过期清理
- **实现要点**：定期清理过期的执行标记和状态信息
- **具体逻辑**：
  - 清理过期的执行标记（超过30分钟）
  - 清理过期的任务执行时间标记（超过1小时）
  - 清理过期的运行中任务信息（超过5分钟）
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `onMounted` 函数
  - `BatchDailyTasks.vue` 中的 `cleanupLocalStorageTasks` 函数

#### 3.4 序列化安全
- **实现要点**：处理循环引用和无法序列化的对象
- **具体逻辑**：
  - 序列化前清理可能导致循环引用的属性
  - 限制队列长度，避免存储过大
  - 捕获序列化异常，确保数据安全
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `saveTaskQueueToStorage` 函数

### 4. 分批执行优化

#### 4.1 灵活配置
- **实现要点**：支持自定义批处理大小和延迟时间
- **具体逻辑**：
  - 优先使用定时任务的批处理配置
  - 提供默认值，确保兼容性
- **应用位置**：
  - `BatchDailyTasks.vue` 中的 `startBatch` 函数
  - `BatchDailyTasks.vue` 中的定时任务执行逻辑

#### 4.2 进度显示
- **实现要点**：实时显示批量执行进度
- **具体逻辑**：
  - 计算总批次数
  - 每批执行完成后更新进度
  - 在等待期间保持进度显示
- **应用位置**：
  - `BatchDailyTasks.vue` 中的批处理逻辑

#### 4.3 错误隔离
- **实现要点**：单个批次失败不影响其他批次执行
- **具体逻辑**：
  - 单个批次执行失败后，记录错误并继续执行下一批
  - 错误处理完善，确保系统稳定
- **应用位置**：
  - `BatchDailyTasks.vue` 中的批处理循环

#### 4.4 暂停检测
- **实现要点**：批处理等待期间也会检查暂停状态
- **具体逻辑**：
  - 在批处理等待期间定期检查暂停状态
  - 检测到暂停时，将任务加入积攒队列
- **应用位置**：
  - `BatchDailyTasks.vue` 中的批处理等待逻辑

### 2. 任务去重逻辑

**文件**：`src/stores/batchTaskStore.ts`

**实现**：
```typescript
function addToTaskQueue(task: {
  id?: number;
  name?: string;
  selectedTokens?: string[];
  selectedTasks?: string[];
  runType?: string;
}) {
  // 同时检查任务名称、运行类型、账号列表和任务类型
  const isTaskExists = taskQueue.value.some(existingTask => {
    if (existingTask.name === task.name && existingTask.runType === task.runType) {
      const existingTokens = existingTask.selectedTokens || [];
      const newTokens = task.selectedTokens || [];
      const sameTokens = existingTokens.length === newTokens.length && 
             existingTokens.every(tokenId => newTokens.includes(tokenId));
      
      const existingTasks = existingTask.selectedTasks || [];
      const newTasks = task.selectedTasks || [];
      const sameTasks = JSON.stringify(existingTasks.sort()) === JSON.stringify(newTasks.sort());
      
      return sameTokens && sameTasks;
    }
    return false;
  });
  
  if (!isTaskExists) {
    taskQueue.value.push(task);
    console.log('[BatchTask] 任务已加入队列:', task.name);
  } else {
    console.log('[BatchTask] 任务已在队列中，跳过重复添加:', task.name);
  }
}
```
