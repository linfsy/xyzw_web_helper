# 定时任务调度器全局化方案

## 一、问题背景

### 1.1 当前问题
你在使用APK时发现：
- 安装APK后停留在首页（图一画面），不在批量日常页面
- 定时任务倒计时虽然显示，但实际上定时任务调度器没有在工作
- 只有当你进入批量日常页面后，定时任务才开始重新计算时间
- 这意味着如果你一直在首页或其他页面，到时间的任务不会被执行

### 1.2 根本原因
**定时任务调度器绑定在组件生命周期中**

具体来说：
- 定时任务调度器在 `BatchDailyTasks.vue` 组件的 `onMounted` 钩子中初始化
- 当你离开批量日常页面（切换到首页），组件被销毁，`onBeforeUnmount` 钩子清理了所有定时器
- 当你返回批量日常页面，组件重新创建，定时器重新初始化
- 这就是为什么你感觉"不在批量日常页面他就不做任务了"

### 1.3 代码位置
问题代码在 `src/views/BatchDailyTasks.vue`：
```javascript
// 第 6108 行左右
onBeforeUnmount(() => {
  if (countdownInterval) {
    clearInterval(countdownInterval)
    countdownInterval = null
  }
  // ... 清理其他定时器
})
```

## 二、解决方案

### 2.1 核心思路
**将定时任务调度器从"组件级别"提升到"应用级别"**

类比理解：
- **组件级别**：像房间里的灯，离开房间就关灯
- **应用级别**：像家里的总电源，只要家里有人就一直供电

### 2.2 技术方案
使用 **Pinia Store + App.vue 初始化** 的组合方案：

```
┌─────────────────────────────────────┐
│           App.vue (应用入口)          │
│  ┌───────────────────────────────┐  │
│  │     初始化全局定时任务调度器      │  │
│  │   (应用启动时启动，永不停止)      │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│    scheduledTaskStore (Pinia Store)  │
│  ┌───────────────────────────────┐  │
│  │     管理定时任务所有状态        │  │
│  │  - 任务列表                    │  │
│  │  - 倒计时                      │  │
│  │  - 定时器                      │  │
│  │  - 执行逻辑                    │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
                   │
         ┌─────────┴─────────┐
         ▼                   ▼
┌─────────────────┐  ┌─────────────────┐
│ BatchDailyTasks │  │   其他页面       │
│    (批量日常)    │  │  (首页/设置等)   │
│  ┌───────────┐  │  │  ┌───────────┐  │
│  │ 读取状态   │  │  │  │ 读取状态   │  │
│  │ 显示UI    │  │  │  │ 显示倒计时 │  │
│  └───────────┘  │  │  └───────────┘  │
└─────────────────┘  └─────────────────┘
```

### 2.3 为什么选这个方案？

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **Pinia Store** | 简单、Vue官方推荐、易于维护 | 页面关闭后停止 | ⭐⭐⭐⭐⭐ |
| Service Worker | 页面关闭后也能运行 | 复杂、兼容性差、难调试 | ⭐⭐ |
| 纯 App.vue | 简单 | 逻辑分散、难以维护 | ⭐⭐⭐ |

**选择理由：**
1. **符合项目技术栈**：项目已经使用 Pinia 管理状态
2. **易于理解和维护**：Pinia 是 Vue 官方推荐的状态管理方案
3. **渐进式实施**：可以分阶段实施，降低风险
4. **后续扩展方便**：可以轻松添加更多全局功能

#### 3.1.2 文件内容结构
```typescript
// 1. 导入依赖
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { useLocalStorage } from '@vueuse/core'

// 2. 定义 Store
export const useScheduledTaskStore = defineStore('scheduledTask', () => {
  // ========== 状态定义 ==========
  
  // 定时任务列表（使用 localStorage 持久化）
  const scheduledTasks = useLocalStorage('scheduledTasks_v2', [])
  
  // 下次执行时间（用于显示倒计时）
  const nextExecutionTimes = ref({})
  
  // 调度器运行状态
  const isSchedulerRunning = ref(false)
  
  // 定时器引用（用于清理）
  let countdownInterval = null      // 倒计时定时器（每秒更新）
  let taskCheckInterval = null      // 任务检查定时器（每5秒检查一次）
  
  // ========== 计算属性 ==========
  
  // 获取启用的任务列表
  const enabledTasks = computed(() => {
    return scheduledTasks.value.filter(task => task.enabled)
  })
  
  // 获取即将执行的任务
  const upcomingTasks = computed(() => {
    // 按执行时间排序
    return enabledTasks.value.sort((a, b) => {
      const timeA = nextExecutionTimes.value[a.id] || Infinity
      const timeB = nextExecutionTimes.value[b.id] || Infinity
      return timeA - timeB
    })
  })
  
  // ========== 方法定义 ==========
  
  /**
   * 启动定时任务调度器
   * 说明：应用启动时调用一次，之后一直运行
   */
  const startScheduler = () => {
    // 防止重复启动
    if (isSchedulerRunning.value) {
      console.log('[ScheduledTask] 调度器已在运行')
      return
    }
    
    console.log('[ScheduledTask] 启动定时任务调度器')
    
    // 1. 立即更新一次倒计时
    updateCountdowns()
    
    // 2. 启动倒计时定时器（每秒更新显示）
    countdownInterval = setInterval(() => {
      updateCountdowns()
    }, 1000)
    
    // 3. 启动任务检查定时器（每5秒检查是否有任务到时间）
    taskCheckInterval = setInterval(() => {
      checkAndExecuteTasks()
    }, 5000)
    
    // 4. 标记为运行中
    isSchedulerRunning.value = true
    
    // 5. 记录日志
    console.log('[ScheduledTask] 调度器启动成功')
  }
  
  /**
   * 停止定时任务调度器
   * 说明：应用关闭时调用，清理定时器
   */
  const stopScheduler = () => {
    console.log('[ScheduledTask] 停止定时任务调度器')
    
    // 清理倒计时定时器
    if (countdownInterval) {
      clearInterval(countdownInterval)
      countdownInterval = null
    }
    
    // 清理任务检查定时器
    if (taskCheckInterval) {
      clearInterval(taskCheckInterval)
      taskCheckInterval = null
    }
    
    // 标记为停止
    isSchedulerRunning.value = false
    
    console.log('[ScheduledTask] 调度器已停止')
  }
  
  /**
   * 更新倒计时
   * 说明：每秒执行一次，计算每个任务的剩余时间
   */
  const updateCountdowns = () => {
    const now = new Date()
    const newTimes = {}
    
    enabledTasks.value.forEach(task => {
      // 计算下次执行时间
      const nextTime = calculateNextExecutionTime(task, now)
      newTimes[task.id] = nextTime
    })
    
    // 更新状态（触发 UI 更新）
    nextExecutionTimes.value = newTimes
  }
  
  /**
   * 检查并执行任务
   * 说明：每5秒执行一次，检查是否有任务到时间需要执行
   */
  const checkAndExecuteTasks = () => {
    const now = new Date()
    
    enabledTasks.value.forEach(task => {
      // 检查是否到执行时间
      if (shouldExecuteTask(task, now)) {
        console.log(`[ScheduledTask] 任务到时间，准备执行: ${task.name}`)
        executeTask(task)
      }
    })
  }
  
  /**
   * 执行具体任务
   * 说明：调用 BatchDailyTasks.vue 中的任务执行逻辑
   */
  const executeTask = async (task) => {
    try {
      // 这里需要调用 BatchDailyTasks.vue 中的任务执行方法
      // 可以通过事件总线或全局方法实现
      console.log(`[ScheduledTask] 执行任务: ${task.name}`)
      
      // TODO: 调用任务执行逻辑
      // 方案1：使用事件总线
      // eventBus.emit('executeScheduledTask', task)
      
      // 方案2：使用全局方法
      // window.executeScheduledTask?.(task)
      
      // 方案3：在 BatchDailyTasks.vue 中监听 Store 变化
    } catch (error) {
      console.error(`[ScheduledTask] 任务执行失败: ${task.name}`, error)
    }
  }
  
  /**
   * 计算下次执行时间
   */
  const calculateNextExecutionTime = (task, now) => {
    // 根据任务类型计算
    if (task.runType === 'daily') {
      // 每日任务：今天的执行时间
      const [hours, minutes] = task.runTime.split(':')
      const nextTime = new Date(now)
      nextTime.setHours(parseInt(hours), parseInt(minutes), 0, 0)
      
      // 如果今天的时间已过，算明天的
      if (nextTime < now) {
        nextTime.setDate(nextTime.getDate() + 1)
      }
      
      return nextTime.getTime()
    }
    
    // TODO: 处理 cron 表达式等其他类型
    return Infinity
  }
  
  /**
   * 检查是否应该执行任务
   */
  const shouldExecuteTask = (task, now) => {
    // 获取下次执行时间
    const nextTime = nextExecutionTimes.value[task.id]
    if (!nextTime) return false
    
    // 检查是否到时间（允许5秒的误差）
    const diff = now.getTime() - nextTime
    return diff >= 0 && diff < 5000
  }
  
  // ========== 任务管理方法 ==========
  
  /**
   * 添加任务
   */
  const addTask = (task) => {
    scheduledTasks.value.push({
      ...task,
      id: Date.now(), // 生成唯一ID
      enabled: true,
      createdAt: new Date().toISOString(),
    })
    console.log('[ScheduledTask] 添加任务:', task.name)
  }
  
  /**
   * 删除任务
   */
  const removeTask = (taskId) => {
    const index = scheduledTasks.value.findIndex(t => t.id === taskId)
    if (index > -1) {
      const taskName = scheduledTasks.value[index].name
      scheduledTasks.value.splice(index, 1)
      console.log('[ScheduledTask] 删除任务:', taskName)
    }
  }
  
  /**
   * 更新任务
   */
  const updateTask = (taskId, updates) => {
    const task = scheduledTasks.value.find(t => t.id === taskId)
    if (task) {
      Object.assign(task, updates)
      console.log('[ScheduledTask] 更新任务:', task.name)
    }
  }
  
  /**
   * 启用/禁用任务
   */
  const toggleTask = (taskId) => {
    const task = scheduledTasks.value.find(t => t.id === taskId)
    if (task) {
      task.enabled = !task.enabled
      console.log('[ScheduledTask] 切换任务状态:', task.name, task.enabled)
    }
  }
  
  // ========== 返回公开接口 ==========
  return {
    // 状态
    scheduledTasks,
    nextExecutionTimes,
    isSchedulerRunning,
    
    // 计算属性
    enabledTasks,
    upcomingTasks,
    
    // 方法
    startScheduler,
    stopScheduler,
    addTask,
    removeTask,
    updateTask,
    toggleTask,
  }
})
```

#### 3.1.3 关键设计说明

**为什么用 `useLocalStorage`？**
- 任务列表需要持久化，刷新页面后不丢失
- Pinia 的 `useLocalStorage` 会自动同步状态到 localStorage

**为什么两个定时器？**
- `countdownInterval`（每秒）：更新倒计时显示，需要频繁更新
- `taskCheckInterval`（每5秒）：检查任务是否到时间，不需要太频繁

**为什么用 `ref` 而不是 `reactive`？**
- `ref` 更适合原始值和需要替换的对象
- `reactive` 适合复杂的嵌套对象
- 这里的状态相对简单，用 `ref` 更清晰

#### 3.3.5 保留的任务执行逻辑

**重要：** 任务实际执行逻辑（调用 WebSocket、处理任务队列等）仍然保留在 BatchDailyTasks.vue 中

**为什么？**
- Store 只负责任务调度和状态管理
- 任务执行需要访问组件中的方法和状态（如 WebSocket 连接、任务队列等）
- 通过事件或回调机制让 Store 通知组件执行任务

**实现方案：**

方案1：事件总线（推荐）
```typescript
// 创建事件总线（utils/eventBus.js）
import mitt from 'mitt'
export const eventBus = mitt()

// Store 中触发事件
import { eventBus } from '@/utils/eventBus'

const executeTask = (task) => {
  eventBus.emit('executeScheduledTask', task)
}

// BatchDailyTasks.vue 中监听事件
import { eventBus } from '@/utils/eventBus'

onMounted(() => {
  eventBus.on('executeScheduledTask', (task) => {
    // 执行任务
    manualExecuteTask(task)
  })
})

onUnmounted(() => {
  eventBus.off('executeScheduledTask')
})
```

方案2：回调函数
```typescript
// Store 中接受回调
const executeTask = async (task, executor) => {
  if (executor) {
    await executor(task)
  }
}

// BatchDailyTasks.vue 中注册执行器
onMounted(() => {
  scheduledTaskStore.setTaskExecutor(manualExecuteTask)
})
```

## 四、风险及应对措施

### 4.1 风险清单

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 定时器重复启动 | 任务重复执行 | 中 | Store 中使用标志位检查 |
| 状态不同步 | 页面显示错误 | 中 | 使用 computed 从 Store 读取 |
| 任务不执行 | 功能失效 | 低 | 检查事件监听是否正确 |
| 内存泄漏 | 应用卡顿 | 低 | 确保定时器正确清理 |
| 数据丢失 | 任务列表消失 | 低 | 使用 useLocalStorage 持久化 |

### 4.2 常见问题排查

**问题1：定时器重复启动**
- 现象：控制台多次显示"调度器启动成功"
- 原因：没有检查是否已在运行
- 解决：确保 `startScheduler` 中有 `if (isSchedulerRunning.value) return`

**问题2：任务不执行**
- 现象：到时间了任务没有执行
- 原因：事件监听没有正确设置
- 解决：检查 `eventBus.on` 是否在 `onMounted` 中，以及 `eventBus.off` 是否在 `onUnmounted` 中

**问题3：倒计时不更新**
- 现象：倒计时显示为静态数字
- 原因：没有正确从 Store 获取状态
- 解决：确保使用 `computed(() => scheduledTaskStore.nextExecutionTimes)`

**问题4：页面切换后任务状态丢失**
- 现象：切换页面后任务列表为空
- 原因：Store 没有正确持久化
- 解决：确保使用 `useLocalStorage('scheduledTasks_v2', [])`
