## 3. 详细实施方案

### 3.1 优雅任务暂停（2.2）

**功能描述**：
暂停时将任务加入积攒队列，保留执行状态，暂停时间结束后自动恢复执行。

**实现逻辑**：
1. 检测到暂停状态时，将当前任务加入积攒队列
2. 保留任务的所有配置信息
3. 暂停时间结束后自动恢复执行

**涉及文件**：
- `src/views/BatchDailyTasks.vue` - 任务执行逻辑
- `src/views/BatchDailyTasks.vue` - 定时任务调度器

**代码实现**：

**文件1：src/views/BatchDailyTasks.vue（任务执行逻辑）**
```javascript
// 在任务执行前检查暂停状态
if (isPauseTime.value.paused) {
  addLog({
    time: new Date().toLocaleTimeString(),
    message: `=== 任务被暂停: 当前处于${isPauseTime.value.reason}，已加入积攒队列 ===`,
    type: "info",
  });
  
  // 将任务加入积攒队列
  batchTaskStore.addToTaskQueue({
    name: taskName,
    selectedTokens: selectedTokens.value,
    selectedTasks: [taskFunctionName],
    runType: 'manual',
    id: Date.now()
  });
  
  return;
}
```

**文件2：src/views/BatchDailyTasks.vue（定时任务调度器）**
```javascript
// 定时任务调度器中检查暂停状态
if (shouldRun) {
  if (isPauseTime.value.paused) {
    // 处于暂停时间，加入积攒队列
    const existingTaskIndex = batchTaskStore.taskQueue.findIndex(t => {
      const sameName = t.name === task.name;
      const sameTasks = JSON.stringify(t.selectedTasks?.sort()) === JSON.stringify(task.selectedTasks?.sort());
      const sameTokens = JSON.stringify(t.selectedTokens?.sort()) === JSON.stringify(task.selectedTokens?.sort());
      return sameName && sameTasks && sameTokens;
    });
    
    if (existingTaskIndex === -1) {
      addLog({
        time: currentTime,
        message: `=== 定时任务 ${task.name} 到点但处于${isPauseTime.value.reason}，加入积攒队列 ===`,
        type: "info",
      });
      
      batchTaskStore.addToTaskQueue({
        id: task.id,
        name: task.name,
        selectedTokens: task.selectedTokens,
        selectedTasks: task.selectedTasks,
        runType: task.runType,
        batchSize: task.batchSize,
        batchDelay: task.batchDelay,
        enableBatchExecution: task.enableBatchExecution
      });
    }
    
    continue;
  }
  
  // 正常执行任务
  await executeScheduledTask(task);
}
```

**测试方法**：
1. 设置定时任务在暂停时间段内执行
2. 验证任务被加入积攒队列
3. 等待暂停时间结束
4. 验证任务自动执行

**注意事项**：
- 保留任务的完整配置信息
- 避免重复添加相同的任务
- 添加日志记录

### 3.2 暂停检测（4.4）

**功能描述**：
批处理等待期间也会检查暂停状态，检测到暂停时，将任务加入积攒队列。

**实现逻辑**：
1. 在批处理等待期间定期检查暂停状态
2. 检测到暂停时，将任务加入积攒队列
3. 保留任务的执行状态

**涉及文件**：
- `src/views/BatchDailyTasks.vue` - 批处理等待逻辑

**代码实现**：

**文件：src/views/BatchDailyTasks.vue**
```javascript
// 批处理等待期间检查暂停状态
if (batchIndex < totalBatches - 1 && !batchTaskStore.shouldStop.value) {
  addLog({
    time: new Date().toLocaleTimeString(),
    message: `=== 第 ${batchIndex + 1} 批执行完成，等待 ${batchDelay / 1000} 秒后执行下一批 ===`,
    type: "info",
  });

  let remainingSeconds = batchDelay / 1000;
  
  while (remainingSeconds > 0 && !batchTaskStore.shouldStop.value) {
    // 检查暂停状态
    if (isPauseTime.value.paused) {
      addLog({
        time: new Date().toLocaleTimeString(),
        message: `=== 批量任务被暂停: 当前处于${isPauseTime.value.reason}，已加入积攒队列 ===`,
        type: "info",
      });
      
      // 将剩余任务加入积攒队列
      const remainingTokens = availableTokens.slice((batchIndex + 1) * batchSize);
      batchTaskStore.addToTaskQueue({
        name: task.name,
        selectedTokens: remainingTokens,
        selectedTasks: taskTasksList,
        runType: task.runType,
        id: task.id
      });
      
      selectedTokens.value = [...originalSelectedTokens];
      return;
    }
    
    await new Promise(resolve => setTimeout(resolve, 1000));
    remainingSeconds--;
  }
}
```

**测试方法**：
1. 启动批量任务
2. 在等待期间设置暂停时间
3. 验证任务被暂停并加入积攒队列
4. 等待暂停时间结束
5. 验证任务自动恢复执行

**注意事项**：
- 暂停检测要实时
- 确保任务状态被正确保存

### 3.3 错误隔离（4.3）

**功能描述**：
单个批次失败不影响其他批次执行，错误处理完善，确保系统稳定。

**实现逻辑**：
1. 单个批次执行失败后，记录错误并继续执行下一批
2. 错误处理完善，确保系统稳定
3. 添加详细的错误日志

**涉及文件**：
- `src/views/BatchDailyTasks.vue` - 批处理循环

**代码实现**：

**文件：src/views/BatchDailyTasks.vue**
```javascript
// 批处理循环中的错误处理
for (let batchIndex = 0; batchIndex < totalBatches; batchIndex++) {
  if (batchTaskStore.shouldStop.value) break;
  
  const startIndex = batchIndex * batchSize;
  const endIndex = Math.min(startIndex + batchSize, totalTokens);
  const batchTokens = availableTokens.slice(startIndex, endIndex);
  
  try {
    addLog({
      time: new Date().toLocaleTimeString(),
      message: `=== 开始执行第 ${batchIndex + 1}/${totalBatches} 批，共 ${batchTokens.length} 个账号 ===`,
      type: "info",
    });
    
    selectedTokens.value = batchTokens;
    
    // 执行批次任务
    for (const taskName of taskTasksList) {
      if (batchTaskStore.shouldStop.value) break;
      
      const taskFunction = getOriginalTaskFunction(taskName);
      if (typeof taskFunction === 'function') {
        try {
          await taskFunction();
          batchTaskStore.resetShouldStop();
        } catch (error) {
          // 单个任务失败，记录错误但继续执行
          addLog({
            time: new Date().toLocaleTimeString(),
            message: `批次 ${batchIndex + 1} 中任务 ${taskName} 执行失败: ${error.message}，继续执行下一批`,
            type: "error",
          });
          console.error(`批次 ${batchIndex + 1} 任务执行失败:`, error);
        }
      }
    }
    
    // 更新进度
    batchTaskStore.setProgress(Math.round(((batchIndex + 1) / totalBatches) * 100));
    
  } catch (batchError) {
    // 整个批次失败，记录错误但继续执行下一批
    addLog({
      time: new Date().toLocaleTimeString(),
      message: `第 ${batchIndex + 1} 批执行失败: ${batchError.message}，继续执行下一批`,
      type: "error",
    });
    console.error(`批次 ${batchIndex + 1} 执行失败:`, batchError);
  }
  
  // 等待下一批
  if (batchIndex < totalBatches - 1 && !batchTaskStore.shouldStop.value) {
    await new Promise(resolve => setTimeout(resolve, batchDelay));
  }
}
```

**测试方法**：
1. 模拟某个批次执行失败
2. 验证后续批次继续执行
3. 验证错误被正确记录

**注意事项**：
- 错误处理要完善
- 确保系统稳定性
- 添加详细的错误日志

### 5.1 风险识别
