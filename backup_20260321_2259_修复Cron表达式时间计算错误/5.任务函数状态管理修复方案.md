# 任务函数状态管理修复方案

## 问题描述

任务函数（如 batchLegacyClaim、batchLegacyGiftSendEnhanced 等）内部直接调用了 `startTask()` 和 `stopTask()`，导致：

1. 当从积攒队列执行任务时，任务函数内部调用 `stopTask()` 将 `isRunning` 设为 false
2. 积攒队列的下一个任务执行时，`isRunning` 已经是 false 了
3. 形成循环调用，导致任务重复加入积攒队列

## 当前修复方案（短期方案）

在 `executeInBatches` 函数中保存/恢复 `isRunning` 状态：

```javascript
// 重要：保存当前的 isRunning 状态
const wasRunningBeforeTask = batchTaskStore.isRunning;

try {
  await taskFunction();
} finally {
  // 重要：恢复 isRunning 状态
  if (wasRunningBeforeTask && !batchTaskStore.isRunning) {
    batchTaskStore.startTask();
  }
}
```

**优点：**
- 一处修改，全局生效
- 不需要修改所有任务文件
- 向后兼容

**缺点：**
- 任务函数内部仍有冗余代码
- 是一种"补丁"方案

## 长期方案（逐步实施）

### 目标
逐步修改所有任务文件，移除内部的 `startTask()` 和 `stopTask()` 调用。

### 修改步骤

#### 步骤1：定位 startTask 调用
在每个文件中搜索：
```javascript
if (startTask) startTask();
else { startTask(); }
```

替换为：
```javascript
// 移除 startTask 调用，由调用方管理 isRunning 状态
```

#### 步骤2：定位 stopTask 调用
在每个文件中搜索：
```javascript
if (stopTask) stopTask();
else { stopTask(); }
```

替换为：
```javascript
// 移除 stopTask 调用，由调用方管理 isRunning 状态
```

#### 步骤3：添加注释说明
在函数注释中添加：
```javascript
/**
 * 函数名
 * 注意：此函数不管理 isRunning 状态，由调用方（executeInBatches）管理
 */
```

#### 步骤4：测试验证
修改每个文件后，执行以下测试：
1. 单独执行该任务，验证是否正常
2. 从积攒队列执行该任务，验证是否正常
3. 检查日志中是否有 "检测到任务函数改变了 isRunning 状态" 的日志（应该不再有）

### 修改示例

**修改前：**
```javascript
const batchLegacyClaim = async () => {
  if (selectedTokens.value.length === 0) return;
  if (startTask) startTask();
  else { startTask(); }

  // ... 任务逻辑 ...

  if (stopTask) stopTask();
  else { stopTask(); }
};
```

**修改后：**
```javascript
/**
 * 批量领取功法残卷
 * 注意：此函数不管理 isRunning 状态，由调用方（executeInBatches）管理
 */
const batchLegacyClaim = async () => {
  if (selectedTokens.value.length === 0) return;
  // 移除 startTask 调用，由调用方管理 isRunning 状态

  // ... 任务逻辑 ...

  // 移除 stopTask 调用，由调用方管理 isRunning 状态
};
```

## 验证方法

### 1. 检查日志
修改后，日志中不应该再出现：
```
=== 检测到任务函数改变了 isRunning 状态，已恢复 ===
```

如果出现，说明该任务文件还没有修改。

### 2. 功能测试
测试以下场景：
1. 单独执行收残卷任务
2. 单独执行送残卷任务
3. 先执行收残卷，再立即执行送残卷（测试积攒队列）
4. 定时任务到期执行

### 3. 代码检查
检查任务文件中是否还有：
- `startTask()` 调用
- `stopTask()` 调用

## 回滚方案

如果修改后出现问题，可以：
1. 从备份恢复文件
2. 或者重新添加 `startTask/stopTask` 调用

## 注意事项

1. **不要一次性修改所有文件**：建议逐个修改，每修改一个就测试一个
2. **保留注释**：修改后的注释有助于后续维护
3. **检查依赖**：确保任务函数不再依赖 `startTask/stopTask` 的副作用
4. **测试充分**：修改后必须测试单独执行和积攒队列执行两种场景
