## [2026-03-11] - 优化账号连接重试功能

### 文件：`src/views/BatchDailyTasks.vue`

#### 修改类型（优化）

**问题描述：**
- 账号连接失败时，系统设置了 maxRetries = 2，但实际只重试了 1 次
- 重试逻辑使用简单的 if 判断，无法实现真正的多次重试

**优化方案：**
- 将重试逻辑从简单的 if 判断改为 while 循环
- 实现真正的 maxRetries 次重试
- 改进重试日志，显示重试次数
- 确保每次重试都使用最新的 token 信息

**关键代码：**
```javascript
// 优化前：简单的 if 判断，只重试 1 次
if (!connected && maxRetries > 0) {
  // 尝试重连 1 次
}

// 优化后：使用 while 循环，支持真正的多次重试
let attempts = 0;
while (!connected && attempts <= maxRetries) {
  if (attempts > 0) {
    addLog({
      time: new Date().toLocaleTimeString(),
      message: `连接超时，尝试重连... (${attempts}/${maxRetries})`,
      type: "warning",
    });
    // 等待重连延迟
  }
  // 尝试连接
  connected = await waitForConnection(tokenId);
  attempts++;
}
```

**重试流程：**
1. 首次连接尝试
2. 如果失败，等待重连延迟时间
3. 尝试第 1 次重连
4. 如果仍然失败，等待重连延迟时间
5. 尝试第 2 次重连
6. 如果所有尝试都失败，标记为连接失败

**日志示例：**
```
正在连接... (队列: 1/5)
连接超时，尝试重连... (1/2)
正在重连... (1/2)
连接超时，尝试重连... (2/2)
正在重连... (2/2)
连接失败 (已重试 2 次)
```

**修复方案：**

   - 将 `toggleTaskEnabled` 函数中的 `toggleTask` 改为 `updateTask`
   - 直接使用开关的新状态更新任务，避免状态被切换两次

2. **修复任务执行标记管理**
   - 在任务冲突时添加 `safeLocalStorage.removeItem(`task_executing_${task.id}`)`
   - 将 finally 块中的 `localStorage.removeItem` 改为 `safeLocalStorage.removeItem`
   - 确保任务执行标记在所有情况下都能正确清除

3. **修复语法错误**
   - 删除多余的 try 块，确保所有 try 块都有对应的 catch 或 finally 子句

**关键代码：**

```javascript
// 1. 修复定时任务开关功能
```
