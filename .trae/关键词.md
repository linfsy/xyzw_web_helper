## 使用示例

### 构建APK
```

```
- 执行完整的APK构建流程
- 包括更新版本号、构建Web项目、同步Capacitor、构建debug版APK

### 创建备份
```
创建备份
```
- 自动从修改日志获取最近一次修改的标题作为备份描述
- 创建格式为 `backup_YYYYMMDD_HHmm_描述性后缀` 的备份

### 安全更新
```
安全更新
```
- 执行安全更新流程
- 包括创建备份、分析上游变更、选择性集成、测试验证、更新日志

### 拉取上游更新
```
拉取上游分仓库更新
```

- 使用安全更新策略，不直接覆盖文件

### 运行服务器
```
运行服务器
```
- 启动本地开发服务器
- 通常运行在 http://localhost:3000/ 或 http://localhost:3001/

## 安全更新策略

当使用任何更新相关的关键词时，系统会自动遵循安全更新策略：

1. **不直接覆盖文件**：采用选择性集成，保留所有本地功能
2. **创建备份**：在更新前创建完整备份，确保可回滚
3. **充分测试**：在集成后进行测试，确保所有功能正常运行
4. **详细记录**：更新修改日志，记录所有变更

这样可以确保更新过程的安全性和可靠性，避免因更新导致的BUG和功能丢失。
